## 🔴 Sen qilishing SHART

Men **sirlar (secrets) yoza olmayman**. Quyidagilarni `server/.env` ga o'zing qo'sh:

```bash
# 1) NFC qurilma kaliti — AKS HOLDA /attendance/nfc 503 (o'chiq) qaytaradi!
#    Kalit yaratish:
#    python -c "import secrets; print(secrets.token_hex(32))"
NFC_DEVICE_API_KEY=<yuqoridagi buyruq natijasi>

# 2) Agar Railway / nginx kabi reverse-proxy ORQASIDA bo'lsang (odatda 1):

```

> ⚠️ **NFC reader qurilmasi** endi har so'rovda `X-NFC-Device-Key: <kalit>` header'ini yuborishi kerak.

## 💰 1. Pul (Money)

### 1.1 Manfiy oylik tuzatildi (clamp)
**Muammo:** `final_salary = base + (bonus − avans)` — agar avans bazadan oshsa, oylik **manfiy** bo'lib saqlanardi. Xodim "−1 000 000 to'landi" bildirishnomasini olardi, kompaniya qarzi izsiz yo'qolardi.

**Yechim:** `final_salary = max(0, base + net)`. DB ichida ham `GREATEST(0, ...)`.
- `server/app/services/salary_history_services.py` — `calculate_salary`
- `server/app/repositories/salary_history_repository.py` — `recompute_if_unpaid`

```python
# Oldin:
final_salary = base_salary + net
# Endi:
final_salary = max(Decimal("0"), base_salary + net)
```

### 1.2 Lost-update race tuzatildi (atomik UPDATE)
**Muammo:** Avans qo'shish `get → if not is_paid → o'zgartir → commit` qilardi (read-modify-write). To'lov o'qish bilan yozish orasiga tushib qolsa, **to'langan oylik ustiga eski qiymat yozilardi**. Kassadan 3M chiqib, kitobda 2.5M ko'rinardi.

**Yechim:** Yangi `recompute_if_unpaid` — **bitta atomik shartli UPDATE**: `WHERE is_paid IS FALSE`. To'langan oylikka tegmaydi.
- `server/app/repositories/salary_history_repository.py`

```python
update(SalaryHistory)
  .where(..., SalaryHistory.is_paid.is_(False))   # <- DB o'zi tekshiradi, race yo'q
  .values(final_salary=func.greatest(Decimal("0"), SalaryHistory.base_salary + net))
```

> **Tushuncha (race condition):** Ikki so'rov bir vaqtda kelsa, ularning tartibi
> oldindan ma'lum emas. "Avval o'qib, keyin yozish" o'rtasida boshqa so'rov o'zgartirsa,
> sening o'zgarishing eskirgan ma'lumotga asoslanadi. Atomik UPDATE — o'qish va yozishni
> DB darajasida **bir amalga** birlashtirib, bu teshikni yopadi.

### 1.3 Avans/premiya summasiga yuqori chegara
**Muammo:** `amount` faqat `gt=0` edi. `Numeric(10,2)` ustuni maks `99 999 999.99`. Ortiqcha nol yozilsa DB xato berib, payroll buzilardi.

**Yechim:** `le=99999999.99` + izoh uzunligi chegarasi.
- `server/app/schemas/salary_adjustment.py`

### 1.4 TOCTOU himoyasi (orphaned avans)
**Muammo:** Race holatida avans yaratilib, lekin oylik o'sha vaqtda to'langan bo'lib qolsa, avans "berilgan" ko'rinardi-yu, oylikdan ushlanmasdi.

**Yechim:** `recompute_if_unpaid` 0 qaytarsa va oylik to'langan bo'lsa — avans satri o'chiriladi (yangi `adj_repo.delete`) va 409 qaytariladi.
- `server/app/services/salary_history_services.py` — `give_adjustment`
- `server/app/repositories/salary_adjustment_repository.py` — yangi `delete`

### 1.5 O'lik `apply_bonus` kodi olib tashlandi
**Muammo:** `apply_bonus` hech qayerda ishlatilmasdi (o'lik kod), lekin u itemli tizimdan boshqacha mantiqqa ega edi — kelajakda chalkashlik xavfi.

**Yechim:** `recompute_if_unpaid` bilan almashtirildi.

## ⏱️ 2. Davomat (Attendance)

### 2.1 Cheksiz soat tuzatildi (smena clamp)
**Muammo:** Xodim chiqishni unutsa, smena bir necha kun "ochiq" qolib, 100+ soatga aylanardi va shu × stavka to'lanardi.

**Yechim:** Har smena `MAX_SHIFT_HOURS = 16` bilan cheklanadi. Excused-kun o'rtachasi ham.
- `server/app/services/salary_history_services.py`

```python
shift_hours = Decimal(seconds) / Decimal(3600)
worked_hours += min(shift_hours, self.MAX_SHIFT_HOURS)   # <- clamp
```

### 2.2 Rahbar davomat tahriri validatsiyasi
**Muammo:** Rahbar `check_in=2020-yil, check_out=hozir` qo'yib, yuz minglab soat = cheksiz pul yaratishi mumkin edi.

**Yechim:** (a) vaqtlar kelajakda emas (5 daq. skew), (b) davomiylik `MAX_SHIFT_HOURS` dan oshmasin.
- `server/app/services/attendance_services.py` — `update`

### 2.3 Yarim tunni kesib o'tuvchi double check-in
**Muammo:** Kunlik limit `work_date`ga bog'liq edi. Tungi smena 00:05'da chiqib qayta kirsa, yangi kun count=0 → ikkinchi to'lov smenasi ochilardi.

**Yechim:** `MIN_REST_HOURS = 6` — oxirgi chiqishdan keyin minimal dam (kalendardan mustaqil). Yangi `get_last_checkout` metodi.
- `server/app/services/attendance_services.py` — `check_in`
- `server/app/repositories/attendance_repository.py` — yangi `get_last_checkout`

## 🔐 3. Auth / Sirlar

### 3.1 NFC endpoint himoyalandi 🔴 (eng kritik)
**Muammo:** `POST /attendance/nfc` hech qanday auth talab qilmasdi. Har kim nfc_uid'ni bilib, istalgan joydan soxta davomat = soxta oylik yaratardi.

**Yechim:** `verify_nfc_device` dependency — `X-NFC-Device-Key` maxfiy kalit, **constant-time** solishtiruv (`secrets.compare_digest`, timing-attack himoyasi). Kalit sozlanmagan bo'lsa endpoint 503 (o'chiq).
- `server/app/core/dependencies.py` — yangi `verify_nfc_device`
- `server/app/routes/attendance_router.py`
- `server/app/core/config.py` — yangi `NFC_DEVICE_API_KEY`

### 3.2 Rate-limiter X-Forwarded-For himoyasi
**Muammo:** Limiter `X-Forwarded-For` ning birinchi qiymatiga ko'r-ko'rona ishonardi. Hujumchi har so'rovda boshqa soxta IP yozib, login brute-force limitini aylanib o'tardi.

**Yechim:** `TRUSTED_PROXY_COUNT` sozlamasi. 0 → faqat haqiqiy TCP IP; n → o'ngdan n-inchi (proxy qo'ygan, soxtalashtirib bo'lmaydigan) IP.
- `server/app/core/rate_limit.py`, `server/app/core/config.py`

### 3.3 JWT — exp/sub/type majburiy
**Muammo:** `jwt.decode` exp claim'ni faqat mavjud bo'lsa tekshirardi. exp'siz token (kelajak xato/boshqa issuer) abadiy qabul qilinardi.

**Yechim:** `options={"require": ["exp", "sub", "type"]}`.
- `server/app/core/security.py`

### 3.4 bcrypt 72-byte muammosi
**Muammo:** bcrypt 5.x da 72 baytdan uzun parol `ValueError` (500) berardi, sxemalar esa 128/200 belgigacha qabul qilardi.

**Yechim:** Parol har doim 72 baytga qisqartiriladi (`_bcrypt_bytes`).
- `server/app/core/security.py`

### 3.5 `.dockerignore` (sirlar himoyasi)
**Muammo:** `.dockerignore` yo'q edi va `Dockerfile` da `COPY . .` — demak `DEBUG=true` bo'lgan `.env` image'ga "pishib" qolardi.

**Yechim:** Yangi `server/.dockerignore` — `.env`, keshlar, `certs/` chiqarib tashlandi.

## 🛂 4. Authorization

### 4.1 `require_role` blanket bypass olib tashlandi
**Muammo:** `require_role` rahbarni **har doim** o'tkazib yuborardi (`if role == leader: return`). Demak `require_role(Role.employee)` rahbar uchun ishlamasdi — kelajakda xodim-only endpoint qo'shilsa, rahbar jimgina kirardi.

**Yechim:** Faqat ruxsat etilgan rollar tekshiriladi.
- `server/app/core/dependencies.py`

> ⚠️ **XULQ O'ZGARISHI:** Endi rahbar "xodim-only" endpointlarga (`/scan`, `/attendance/me`,
> `/leave-request/me`) kira olmaydi. Bu **to'g'ri** — rahbarning `Employee` yozuvi yo'q,
> shuning uchun bu endpointlar rahbar uchun baribir ishlamasdi (service `get_current_employee`
> da xato berardi). Endi himoya ikki qatlam: router **va** service.

### 4.2 Push unsubscribe ownership
**Muammo:** `delete_by_endpoint(endpoint)` — `user_id`ni e'tiborga olmasdi. Boshqa user'ning endpoint'ini bilgan kishi uning push-obunasini o'chira olardi.

**Yechim:** `WHERE`ga `user_id` qo'shildi — har kim faqat o'zinikini o'chiradi.
- `server/app/repositories/push_subscription_repository.py`, `server/app/routes/push_router.py`
